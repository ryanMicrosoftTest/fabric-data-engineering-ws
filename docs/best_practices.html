👣 Bronze “As-Is” Data

Keep Bronze data as raw as possible. Microsoft’s guidance is to preserve original format in Bronze whenever feasible (e.g. landing raw files), only converting to Delta/Parquet if needed. This ensures the source data is captured verbatim for auditability and reprocessing.

⚡ Delta Tables for Performance

Loading directly to Delta tables yields immediate benefits like ACID transactions, schema enforcement, time-travel versioning, and compressed storage. This can boost query performance and reliability in Bronze, at the cost of skipping a raw “backup” copy.

🔁 Hybrid Ingestion Pattern

Many teams use both: land raw files and load them to a Bronze table. This dual approach provides an immutable raw backup (files) and a queryable Delta dataset. It improves troubleshooting and governance (raw files for inspection) while leveraging Delta for speed in downstream steps.
